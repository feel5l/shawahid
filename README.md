# نظام توثيق شواهد الأداء الوظيفي

منصة إلكترونية متكاملة لإدارة وتوثيق المؤشرات المهنية والشواهد الوظيفية للمعلمين والمشرفين التربويين، مبنية باستخدام React و Express مع TypeScript.

## 🎯 نظرة عامة عن المشروع

نظام توثيق شواهد الأداء الوظيفي هو تطبيق ويب شامل مصمم خصيصاً للبيئة التعليمية السعودية، يهدف إلى تسهيل عملية توثيق وإدارة الأداء المهني للمعلمين. يوفر النظام منصة رقمية متكاملة تمكن المعلمين من:

- **توثيق المؤشرات المهنية**: إنشاء وإدارة المؤشرات المهنية المختلفة بطريقة منظمة ومنهجية
- **إدارة المعايير**: تحديد معايير الأداء وتتبع إنجازها بشكل تفصيلي
- **رفع الشواهد**: إضافة وتنظيم الشواهد والمستندات الداعمة للأداء المهني
- **نظام الاعتماد**: تقديم المؤشرات للمدير للمراجعة والاعتماد
- **اختيار الاستراتيجيات**: ربط الأداء بالاستراتيجيات التعليمية المعتمدة

كما يوفر للمديرين:
- **لوحة تحكم شاملة**: عرض إحصائيات المدرسة والأداء العام
- **مراجعة الأداء**: مراجعة واعتماد أو رفض المؤشرات المقدمة من المعلمين
- **تتبع التقدم**: متابعة أداء جميع المعلمين في المدرسة

## 🔍 المميزات

- واجهة مستخدام حديثة وسهلة الاستخدام بتصميم RTL كامل
- نظام أدوار متعدد (معلم، مدير، مشرف، منشئ الموقع)
- نظام مصادقة آمن مع Replit OAuth
- قاعدة بيانات Neon (PostgreSQL Serverless)
- سير عمل اعتماد متقدم للمؤشرات
- إحصائيات وتقارير شاملة
- دعم الوضع الليلي والنهاري
- مكونات UI متقدمة من Radix UI

## 📋 المتطلبات

- Node.js 18 أو أحدث
- قاعدة بيانات Neon (PostgreSQL Serverless)
- npm أو pnpm

```bash

npm run dev

npm run build
```

## 🔧 متغيرات البيئة

قم بإنشاء ملف `.env` في جذر المشروع:

```env
DATABASE_URL=postgresql://user:password@host:port/database?sslmode=require
SESSION_SECRET=your-secret-key
NODE_ENV=production
PORT=3000

# للمصادقة باستخدام OpenID Connect (اختياري)
OPENID_CLIENT_ID=your-client-id
OPENID_CLIENT_SECRET=your-client-secret
OPENID_ISSUER=https://your-issuer.com
```

## 🚀 النشر على Netlify (موصى به للواجهات الأمامية والخلفية المدمجة) ⭐⭐⭐

لقد تم تهيئة هذا المشروع ليعمل بكفاءة عالية على منصة Netlify باستخدام `netlify/functions` لدعم الواجهة الخلفية (Express) مع الواجهة الأمامية (React).

### الخطوات:

1. **ربط المستودع**: قم بزيارة [Netlify](https://netlify.com) واربط مستودع GitHub الخاص بالمشروع.
2. **إعدادات البناء (Build Settings)**:
   - **Build command**: `npm run build`
   - **Publish directory**: `dist/public`
   - **Functions directory**: `netlify/functions`
3. **متغيرات البيئة (Environment Variables)**:
   من لوحة تحكم Netlify (Site settings > Environment variables)، يجب إضافة المتغيرات التالية:
   - `DATABASE_URL`: رابط الاتصال بقاعدة بيانات Postgres (مثل Neon) **يجب أن ينتهي بـ `?sslmode=require`**. ولحل مشكلة الاتصال البطيئة (Infinite Loading)، يُنصح بالاعتماد على رابط برك الاتصال (Pooler URL) وليس الرابط المباشر.
   - `SESSION_SECRET`: مفتاح أمان الجلسات (مثال: `zayd-ibn-thabit-super-secret-2026`).
   - `ADMIN_PASSWORD`: كلمة المرور الخاصة بدخول المدير (مثال: `Zayd12345`).
   - `CREATOR_PASSWORD`: كلمة المرور الخاصة بمنشئ النظام.
4. **التوجيه الديناميكي (Dynamic Routing)**: المشروع يعتمد الآن على تحويل المتصفح التلقائي غير المصرح لهم بالدخول إلى المسارات المحمية مثل `/principal` إلى واجهة المصادقة والتسجيل `/auth` بصورة أنيقة بدل ظهور رسالة `404 Not Found`.

## 🗄️ قاعدة البيانات - Neon (موصى به) ⭐⭐⭐

### لماذا Neon؟
- PostgreSQL Serverless
- 3 GB تخزين مجاني
- سريع جداً
- نسخ احتياطية تلقائية
- دعم Branching للقواعد
- مثالي للتطبيقات الحديثة

### الخطوات:

1. قم بزيارة [Neon.tech](https://neon.tech)
2. أنشئ حساب مجاني
3. أنشئ مشروع جديد
4. انسخ **Connection String**
5. استخدمه في `DATABASE_URL`

**مثال على Connection String:**
```
postgresql://username:password@ep-xxx.neon.tech/neondb?sslmode=require
```

## 👥 نظام الأدوار

النظام يدعم أربعة أدوار رئيسية:

1. **منشئ الموقع (Creator)**: صلاحيات كاملة لإدارة المستخدمين وتعيين الأدوار
2. **مدير المدرسة (Admin/Principal)**: مراجعة واعتماد المؤشرات، عرض إحصائيات المدرسة
3. **مشرف (Supervisor)**: للتوسع المستقبلي لإشراف الأقسام
4. **معلم (Teacher)**: إنشاء وإدارة المؤشرات والشواهد الخاصة

## 📱 دليل الاستخدام

### للمعلمين:

1. **تسجيل الدخول**: 
   - عبر Replit OAuth (Google, GitHub)
   - أو أدخل اسمك مباشرة للدخول السريع
2. **إنشاء مؤشر جديد**: انقر على "إضافة مؤشر جديد" وأدخل العنوان والوصف
3. **إضافة المعايير**: حدد معايير الأداء للمؤشر
4. **إكمال المعايير**: قم بتأشير المعايير عند إنجازها
5. **إضافة الشواهد**: ارفع الشواهد والمستندات الداعمة
6. **التقديم للاعتماد**: عند اكتمال جميع المعايير، قدم المؤشر للمدير
7. **متابعة الحالة**: تابع حالة اعتماد مؤشراتك

### للمديرين:

1. **الدخول للوحة التحكم**: 
   - عبر Replit OAuth (Google, GitHub)
   - أو استخدم دور المدير مع كلمة المرور المخصصة
   - يتم توجيهك تلقائياً للوحة المدير
2. **مراجعة الإحصائيات**: عرض عدد المعلمين والمؤشرات والطلبات المعلقة
3. **مراجعة المؤشرات**: اضغط على المؤشرات المعلقة للمراجعة
4. **الاعتماد أو الرفض**: اعتمد المؤشر مع ملاحظات أو ارفضه مع بيان السبب

## 🗃️ نموذج البيانات

النظام يستخدم 10 جداول رئيسية:

- **users**: معلومات المستخدمين والأدوار
- **sessions**: إدارة الجلسات
- **indicators**: المؤشرات المهنية
- **criteria**: معايير الأداء لكل مؤشر
- **witnesses**: الشواهد والمستندات
- **strategies**: الاستراتيجيات التعليمية (18 استراتيجية)
- **capabilities**: القدرات التعليمية (12 قدرة)
- **changes**: التغييرات التعليمية (12 تغيير)
- **userStrategies**: ربط الاستراتيجيات بالمستخدمين
- **signatures**: طلبات الاعتماد وحالتها
