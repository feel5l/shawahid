   * Right Side: Upload area (File input with a dashed border UI).

 * Ensure dir="rtl" is applied for correct Arabic layout.
Phase 3: Principal UX (The Smart Review Modal)
Create a new component client/src/components/principal/evidence-review-modal.tsx.
Requirements:
 * Accepts props: indicatorTitle, teacherName, fileUrl.
 * Looks up the standard from PERFORMANCE_STANDARDS.

   * Right Side: File Preview area (Placeholder for PDF/Image).

Phase 4: Integration
 * Update client/src/pages/home.tsx (Teacher Dashboard) to use WitnessUploadModal instead of the generic button.

### لماذا ستنجح هذه المطالبة؟
1.  **البيانات المضمنة (Hardcoded Data):** لم أطلب من الوكيل "تخيل" البيانات، بل أعطيته كود `constants.ts` جاهزاً بنسبة 100% يحتوي على نصوص الصور المرفقة. هذا يمنع أي "هلوسة" في المحتوى.
2.  **الفصل المنطقي (Separation of Concerns):** قسمت العمل إلى مراحل (بيانات -> معلم -> مدير -> دمج)، مما يسهل على الوكيل معالجة الكود خطوة بخطوة.
3.  **التوجيه البصري (Visual Direction):** حددت له تخطيط الشبكة (Split View Grid) لضمان ظهور الإرشادات بجانب الملف، وهو الجوهر الهندسي لتحسين تجربة المستخدم.
